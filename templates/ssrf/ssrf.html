SSRF Test Scenarios

Endpoints simulate SSRF evidence and OOB callbacks. Designed for use with SSRForcer.

Query-based

  • OOB confirm Triggers OOB callback — ?url=
  • OOB async Delayed OOB callback — ?url= &delay_ms=
  • Internal IP indicator Returns evidence when internal address detected — ?dest=
  • Cloud metadata indicator Responds to 169.254.169.254 / metadata patterns — ?data=
  • Whitelist + bypass Blocks by default; accepts %2523@ / @%23 bypass
  • Whitelist safe Always blocks — false positive reduction test
  • Whitelist bypass matrix Fragment, userinfo and percent-encoded bypass patterns
  • Encoding/numeric bypass matrix Octal, hex, decimal, IPv6-mapped IP representations
  • safe_no_evidence Always returns ok — expects 0 findings
  • error_safe 500 without connection evidence — false positive test
  • error_internal 500 with internal IP in body — SSRF indicator
  • header_internal Header-injected SSRF — X-Forwarded-Host, X-Original-URL, Referer
